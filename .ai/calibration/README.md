# Calibration Directory

Place per-repo calibration files here. These teach the agents the local
conventions and style of this codebase.

## Suggested files
- `naming.md`     — naming conventions for files, classes, functions
- `patterns.md`   — preferred architectural patterns and anti-patterns
- `boundaries.md` — module / package ownership rules
- `gates.md`      — thresholds for gate checks (complexity, coverage, etc.)
